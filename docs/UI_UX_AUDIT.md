## 一、跨专题全局问题（7 个专题共有）

### #1 顶部指标卡遮挡 3D 场景建筑标签 [P0]

**现象**：7 个专题的顶部 6 个指标卡（每个 ~120px 宽，间距 8px）从 555–1680px 横向覆盖，画面正中恰好是 3D 场景中央建筑群（崇德楼/崇智楼/钟楼/食堂/崇文楼）的标签位置。

| 专题 | 被遮挡标签 |
|------|----------|
| 综合态势 | 崇德楼、崇文楼 |
| 教学研究 | 崇文楼、崇智楼、崇德楼、钟楼 |
| 行政办公 | 崇德楼、崇智楼、崇思楼 |
| 智慧图书 | 崇雅楼、崇文楼、崇思楼 |
| 智慧教学 | 崇文楼、崇德楼、钟楼 |
| 智慧安防 | 崇德楼、崇智楼、崇文楼 |
| 智慧后勤 | 崇文楼、崇德楼、崇思楼 |

**根因**：`OverviewTopMetrics.tsx` 等 7 个 `XxxTopMetrics.tsx` 组件均为 `position: absolute; top: 16px; left: 50%`，与 3D Canvas 标签的 `Html occlude={false}` 同区域。

**优化方案**：
- 方案 A（推荐）：将顶部指标改为**侧边收折**（点击展开），默认只显示 1 行精简指标（4 个数字），不挡 3D
- 方案 B：3D 建筑标签改为 `position-relative` 且只对**选中**建筑显示标签，其他场景下隐藏
- 方案 C：将顶部指标从 6 个减为 3–4 个核心数字（总人数/总班级/总建筑/总占地）

### #2 底部状态栏在浅色模式下几乎不可见 [P0]

**现象**：浅色模式下 `--bottombar-bg: rgba(0,0,0,0.05)` 几乎透明，"智慧校园可视化平台 v0.2.0 | 数据更新: 17:35:46" 等文字混在白底/3D 场景上，对比度 < 1.5:1。

**截图**：`08-logistics-light.png` 右下角"实时连接正常"绿点尚可读，但左下文字基本糊掉。

**根因**：`src/index.css:70` 浅色 `--bottombar-bg: rgba(0,0,0,0.05)` 太透明，未给文字加阴影/对比底。

**优化方案**：
- 浅色 `--bottombar-bg` 改为 `rgba(255,255,255,0.85)` + `backdrop-filter: blur(8px)`
- 或在 `BottomBar.tsx` 文字加 `text-shadow: 0 1px 2px rgba(0,0,0,0.3)`（深色）/ 浅色不需阴影

### #3 教职工组成 (overview-faculty-composition) RingChart 数字冗余 [P1]

**现象**："教职工组成"面板顶部有 3 个 StatCard（196/79/117），下方 RingChart 中心再写"196"，等于"196 + 196/79/117"四重表达同一信息。

**优化方案**：
- 顶部 3 卡精简为 1 大 + 2 小（如：196 大 + ♂79 / ♀117 小字）
- 或删除 RingChart 中心数字、改在右侧加图例
- 建议统一规则：**StatCard 数字 ≠ Chart 中心数字 ≠ 图例文字**，三者各表达不同维度

### #4 资源更新动态 4 个 StatCard 与 资源统计 数字完全重复 [P1]

**现象**：教学研究 专题左侧"资源更新动态"有 4 个 StatCard（6500/3200/1840/48），但上方"资源统计"面板**完全相同的 4 个数字**。读者扫一眼以为是 bug。

**优化方案**：
- 资源更新动态的顶部 4 卡改为**变化量**（"近 7 日新增 +128" / "本周更新 +24"），而非静态累计值
- 或彻底删除 资源更新动态 的 StatCard 区，让"近期更新资源"列表直接占满

### #5 RingChart 中心长文本被裁切 [P0] — **唯一一处功能 bug**

**现象**：智慧教学 → 教学设备 → 设备状态 RingChart 中心文本 "**351/372**" 被裁为 "**051/372**"，首字"3"被左缘切掉。

**截图**：`05-academics.png` 左下角；`components/charts/RingChart.tsx:48` 半径 `['50%', '75%']` → 内圆直径仅占图表 50%。

**根因**：`centerLabel` 为 `graphic.text` 但未测宽度；当文本宽度 > 内圆直径时，graphic 不会自动换行/缩放，左缘被裁。

**优化方案**（`RingChart.tsx`）：
- 半径改为 `['38%', '62%']`（内圆 38% → 直径 76%，可容 7–8 字符）
- 增加 `centerLabelSize` 自适应：文本长度 > 6 时 `fontSize *= 0.6`
- 或 `centerLabel` 支持两行（`{value}\n{subtitle}`）

## 三、跨组件 / 系统性问题

### #6 35 面板的 StatCard 模板不统一

| 变体 | 出现位置 | 字段 |
|------|---------|------|
| A. 3 卡片水平 | 教职工组成、资源统计、出勤排名、教职工考勤、入馆统计 | 大数+单位+标签 |
| B. 4 卡片 2×2 | 借阅统计、课表分布、门禁管理 | 大数+标签 |
| C. 2 卡片 | 访客管理、学生出勤概况 | 大数+趋势↑↓ |
| D. 大数字+标签（独立） | 资产概况、学院 TopMetrics | 单数无卡片背景 |
| E. 大数字 + 副单位 | 学生基础信息（2814 + 52% 48%）、安防态势 | 多指标合并 |

**建议**：统一定义 `<StatCard size="sm|md|lg" value unit trend unitColor>` 组件，让 35 面板从 5 模板收敛到 3 模板。

### #7 BarChart 在 130–150px 高度下文字"挤压"

**现象**：所有 1:1 横向 BarChart（排名类），当高度 < 150px，标签字号 < 12px，文字密度超过 WCAG AA（4.5:1）。

**根因**：`chartTheme.ts:60` `legendFontSize: 11`，但实际 ECharts `axisLabel.fontSize` 直接取 10（`--font-size-xs: 0.7rem` ≈ 10.8px @ 1920px），过小。

**优化方案**：
- `BarChart` 高度 < 150 时自动 `label: { show: false }`，> 150 时才显示数据标签
- 横向 BarChart（排名类）设置 `grid: { left: '25%', right: '8%' }` 固定左间距
- 字体改用 `clamp(10px, 0.7rem, 12px)` 自适应

### #9 Panel 标题栏与 chart 间距过大

### #11 "镇远中学" 3D 文字标签在浅色背景上消失

**现象**：切换到浅色模式后，3D 场景中镇远中学主楼的"镇远中学"四个竖排大字（黑色文字）贴在校名墙上，**背景也是浅黄/浅灰色，对比度 < 2:1**。

**优化方案**：
- 在 `dayNightTheme.ts` 中根据 `uiTheme` 切换文字色：浅色 → `#1a3a5c` 深蓝 / 深色 → `#ffffff`
- 或在 3D 文字上加 `outline` (CSS `text-shadow` 不适用于 3D 文字，改用 material `emissive`)。

### #12 顶部 TopMetrics 6 卡片字号过大

**现象**：6 个数字（48,700㎡ / 88,000㎡ / 60 / 9 / 196 / 2,800）字号 `font-size-xxl: 1.5rem` (24px @ 1920px)，**比 3D 场景中央建筑高亮标签字号 (16px) 大 50%**。

**建议**：TopMetrics 数字降为 `font-size-lg: 1rem` (16px)，保持与 3D 标签的视觉平衡。

### #13 数字滚动动画 (NumberFlip) 在多次 SSE 更新时闪烁

**现象**：右上时间 + 多个 StatCard 数字在每次 SSE 推送（6s）时会重新滚动，造成大屏"抖动"。

**建议**：NumberFlip 增加 `pauseOnEqual` 参数，数字未变不重新动画。

### #14 图表 Legend 与图表主区域距离过近

`chartTheme.ts:30` 默认 `legend.bottom: 0` → Legend 与 X 轴标签在 30px 内重叠。需在 `chartTheme.ts` 调整 `grid.bottom: 30` for legends。

### #15 Panel 右上角折叠按钮与标题距离过近

`DashboardPanel.tsx:79` `▼` 按钮 `padding: '2px 6px'`，与标题文字 `tracking-wide` 几乎贴在一起。

**建议**：折叠按钮加 `marginLeft: auto` 让其紧贴右边缘。

## 七、结论

整体大屏 7 专题 35 面板**已经达到"可演示"水平**（AGENTS.md 248 单元测试 + 80 E2E 全过，0 console error），但与**专业大屏（DataV/阿里云 DataV）** 还有以下差距：

1. **3D 场景与 UI 面板的信息竞争未解决**（P0 #1）
2. **跨面板设计语言不统一**（5 种 StatCard 模板混用，P1 #9）
3. **数据图表在小尺寸下可读性下降**（P1 #10）
4. **浅色/日间模式下 4 处明显问题**（P0 #2/#11）

预计 **P0 全部修复需要 1.5–2 人日**，P1 需要 4–5 人日，P2 1 人日。**合计 ~7 人日**可达到生产级大屏观感。
