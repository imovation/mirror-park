# 智慧校园可视化平台项目需求规格书

---

# 第1章 项目概述

## 1.1 项目背景

学校在日常运营过程中，教务、安防、图书、办公等各类业务系统已积累了大量运行数据，但这些数据分散在各个独立系统中，管理层缺少一个统一的窗口来掌握校园整体运行状况。

本项目计划建设一套智慧校园可视化平台，采用Three.js（WebGL）作为三维渲染引擎，构建校园数字孪生底座，将校园物理空间与各业务系统数据进行融合，在大屏端集中呈现校园运行状态，为管理层提供信息展示与辅助决策工具。

本文档整合合同服务范围、需求分析成果与数据对接清单，作为后续设计、开发、测试与验收的唯一事实源依据。

## 1.2 建设目标

- **构建校园三维数字底座**：对校园主要建筑外观及重点区域室内空间进行数字化还原。
- **建设可视化大屏展示系统**：围绕综合态势、教学研究、行政办公、智慧图书、智慧教学、智慧安防六个专题，集中呈现校园运行数据。
- **实现三维场景与业务数据联动交互**：通过场景点击、数据钻取等方式，提供从全局到局部的多层次信息浏览。

## 1.3 建设范围与边界

### 建设范围

- 校园全局三维场景搭建（含主要建筑及周边设施）
- 重点建筑室内场景搭建（教学楼结构及教室空间）
- 六个专题可视化数据面板开发
- 三维场景与数据面板的交互联动
- 大屏展示效果调优与测试
- 软件部署与1年数据维护

### 边界说明

- 本项目为可视化展示层建设，业务数据由校方已有系统提供，本项目负责数据对接与展示，不涉及业务系统的建设或改造。
- 三维建模范围以本规格书约定的建模区域清单为准。
- 大屏硬件设备由校方提供，本项目负责软件系统的适配与部署。

## 1.4 总体系统架构

系统采用五层架构，自上而下为调用关系：

- **展示层**：大屏可视化展示终端，承载所有专题页面的最终呈现，是用户直接接触的界面层。
- **应用层**：六个业务专题模块——综合态势、教学研究、行政办公、智慧图书、智慧教学、智慧安防，每个专题独立组织数据面板与三维场景内容。
- **引擎层**：分为两个并行的引擎子系统：
  - Three.js三维渲染引擎：负责全局场景、室内场景、场景交互、渲染管线、3D场景标注、3D热力图、镜头路径、氛围渲染等三维相关能力。
  - 数据可视化引擎：负责图表组件库、数字翻牌器、滚动列表、告警弹窗、热力图、仪表盘、卡片轮播、表格列表等数据呈现组件。
  - 两个引擎之间通过场景与数据联动机制实现双向交互。
- **数据对接层**：负责与校方业务系统的数据通信，支持API接口对接（RESTful/WebService）、数据库对接（视图/中间表）、数据文件导入三种方式，并根据数据类型采用实时、准实时、周期、基础四级更新机制。
- **数据源层**：校方已有的各类业务系统，包括人事系统、学生管理系统、教务系统、排课系统、教学资源平台、教研管理系统、OA系统、考勤系统、图书管理系统、借阅系统、视频监控系统、门禁系统、访客系统、食堂管理系统、考试系统、设备管理系统等。

# 第2章 故事线与交互设计

## 2.1 展示逻辑与叙事结构

平台整体采用"全局总览 → 专题聚焦 → 模块钻取 → 详情查看"四层叙事结构：

- **第一层（全局总览）**：以校园三维鸟瞰视角为入口，配合两侧核心运行指标面板，建立对校园全貌的整体认知。
- **第二层（专题聚焦）**：通过导航切换进入六个专题页面，三维场景同步切换至对应的重点区域视角，数据面板呈现该专题下的业务指标。
- **第三层（模块钻取）**：在专题页面内点击具体数据模块或三维场景中的建筑、设备，进入更细粒度的数据呈现。
- **第四层（详情查看）**：以弹窗或侧边栏形式呈现具体数据明细，如设备详情、人员信息、事件记录等。

## 2.2 场景交互流程

平台的场景交互围绕三维场景与数据面板的双向联动展开：

- **场景驱动数据**：在三维场景中点击建筑物、设备点位、区域标注等对象时，数据面板同步更新为该对象的关联数据。
- **数据驱动场景**：在数据面板中点击特定数据项时，三维场景自动定位至对应的空间位置，镜头以动画方式过渡。
- **镜头运动**：各专题页面设定预设镜头路径，进入专题时自动执行镜头动画，从全局视角平滑过渡至专题对应的重点区域。启动后默认以鸟瞰视角展示校园全貌，配合镜头缓慢环绕运动。
- **基础交互**：支持通过点选、缩放、拖拽等鼠标操作进行360度全方位展示，根据需求实现楼栋、房间位置聚焦和跳转。

## 2.3 专题切换设计

| 专题名称 | 三维场景视角 | 场景特征 |
| --- | --- | --- |
| 综合态势 | 校园全景鸟瞰 | 建筑标注、区域划分 |
| 教学研究 | 智慧教室场景 | 教室内部、教学设备 |
| 行政办公 | 行政办公区域 | 办公楼、功能室分布 |
| 智慧图书 | 图书馆场景 | 阅览区域、馆藏空间 |
| 智慧教学 | 教学楼场景 | 教室使用热力、楼层分布 |
| 智慧安防 | 校园全景 | 监控点位、门禁设备标注 |

专题切换时，镜头运动路径经过预先设计与调优，过渡自然流畅。每个专题页面的三维场景配有独立的氛围渲染方案。

# 第3章 视觉设计规范

## 3.1 整体视觉风格

平台视觉风格定位为深色系科技主题，整体色调以深蓝至深灰为主基调，搭配高饱和度的数据色彩，兼顾大屏远距离观看的辨识度与科技氛围。

- **背景层**：深色渐变底色，配合微弱的网格纹理或粒子动效，增强空间层次感。
- **数据层**：蓝、青、绿等色系作为主数据色，橙、红色系作为告警与异常状态色，确保数据信息在深色背景上清晰可读。
- **三维层**：场景以写实偏科技的渲染风格呈现，建筑体量清晰，夜景氛围灯光烘托，与数据面板在视觉上形成统一整体。Three.js引擎的实时渲染能力为场景提供高质量的光影效果与材质表现。

## 3.2 界面布局规范

大屏界面采用"两侧数据面板 + 中央三维场景"布局结构：

- **中央区域**：三维数字孪生场景，占据画面主体。
- **左侧面板**：当前专题的概览性数据与分类统计。
- **右侧面板**：当前专题的明细数据与动态信息。
- **顶部区域**：平台标题、时间信息、专题导航。
- **底部区域**：辅助信息栏或时间轴组件。

各面板区域采用半透明底板设计，与三维场景形成视觉融合。各专题页面遵循统一的布局框架，面板位置与尺寸比例保持一致，仅内容随专题切换而变化。

## 3.3 UI组件与交互设计

平台UI组件遵循统一的设计规范：

- **图表组件**：统一的配色方案、字体规范、坐标轴样式，各类图表采用一致的视觉语言。涵盖折线图、柱状图、饼图、散点图、雷达图、热力图、关系图、树图、矩阵树图、旭日图、桑基图、漏斗图等常规图表，以及根据业务需要从零开发的定制化图表。
- **数据卡片**：核心指标采用数字翻牌器或数字卡片形式，配合图标和单位标注。
- **列表组件**：动态信息采用滚动列表形式，支持自动滚动与悬停暂停。
- **交互反馈**：鼠标悬停、点击等操作提供高亮、缩放、弹出等视觉反馈。
- **弹窗组件**：详情信息以弹窗形式呈现，样式与整体风格统一。

# 第4章 三维场景搭建

## 4.1 室外全局场景

### 4.1.1 建模范围

还原校园主要建筑及周边50米范围内的设施与环境，包括建筑主体、道路、绿化、运动场地、围墙等要素，建模误差不超过1米。场景基于CAD图纸及现场照片进行手工建模，根据展示需要对建筑外立面进行适当贴图处理。

### 4.1.2 建模区域清单

| 序号 | 建模区域 |
| --- | --- |
| 1 | 1号教学、办公楼1区 |
| 2 | 1号教学、办公楼3区、1号教学、办公楼4区 |
| 3 | 1号教学办公楼2区 |
| 4 | 2号食堂、宿舍楼1单元 |
| 5 | 2号食堂、宿舍楼2单元 |
| 6 | 2号食堂、宿舍楼食堂 |
| 7 | 3号体育馆 |
| 8 | 4号门卫室 |
| 9 | 5号钟楼 |

以上建筑均基于竣工图纸（CAD）及现场照片进行建模，周边道路、绿化、地形等环境要素配合场景整体效果搭建。

### 4.1.3 建模精度与技术要求

- **建模精度**：建筑外观轮廓与实际位置偏差不超过1米，建筑体量、层高、开窗等主要特征可辨识。
- **模型制作**：基于CAD竣工图纸进行三维建模，外立面根据展示需要进行贴图处理，贴图参考现场照片。
- **场景优化**：模型面数控制在合理范围内，确保在Three.js引擎中流畅运行。远景建筑适当简化以优化渲染性能，植被、道路等环境要素采用适当的简化处理方式。

## 4.2 室内场景

### 4.2.1 教学楼结构建模（简模）

还原预估五层教学楼的室内墙体结构，基于CAD图纸及现场照片进行手工建模，模型精度为简模级别。

建模范围包括楼层结构、墙体分隔、走廊通道等主要结构要素。不含室内家具设备，不含墙面贴图。模型需清晰反映各楼层的空间布局与房间分布关系。

### 4.2.2 教室室内建模（精模）

对不超过1000平方米的重点房间做室内精模，基于CAD图纸及现场照片进行手工建模，模型精度为精模级别，含室内贴图。

建模范围包括室内墙体结构、地面、天花板、门窗等建筑要素，以及桌椅、讲台、黑板/白板、多媒体设备等主要教学设施。墙面、地面等表面进行贴图处理，体现材质与色彩效果。

## 4.3 模型轻量化处理

对模型进行轻量化处理，包括减面、分层、命名等处理，确保在Three.js引擎中高效渲染。

## 4.4 场景渲染与后期制作

室内场景在Three.js引擎中完成渲染与后期制作：

- **场景搭建**：三维模型导入Three.js后，完成场景组装与空间布局调整。
- **氛围渲染**：根据室内空间特点进行灯光布置与光影调优，利用Three.js的全局光照能力营造真实的室内光照效果，材质表现优化。
- **镜头路径**：绘制室内场景的镜头运动路径，覆盖室内主要空间，运动节奏平稳自然。
- **后期包装**：画面后处理效果调优，配合背景音乐，整体呈现效果与大屏视觉风格保持一致。

# 第5章 定制化数据面板

按照大屏视觉设计稿进行定制化数据面板开发，各面板填入MOCK数据进行整体效果测试，确保系统正常运行与访问。面板支持指标数据在大屏的呈现，支持从仪表和三维场景两个角度对指标进行钻取分析，支持实时数据动态刷新。

## 5.1 综合态势

汇集校园空间、人员、资产、环境等多维数据，呈现学校整体运行状态。中央区域为校园三维全景，两侧面板布置核心运行指标。

### 5.1.1 学校概况

呈现校园基础设施数据，包括占地面积、建筑面积、班级数量等核心指标。三维场景中建筑物支持点击查看基本信息。采用数字翻牌器呈现。

### 5.1.2 人员构成

统计教职工队伍概况，包括教师总数、男女比例、学历分布等。采用数字卡片与环形图组合呈现人员结构。

### 5.1.3 师资分布

反映各学科教师配置情况，以横向柱状图呈现各学科师资力量对比。同时呈现职称分布。

### 5.1.4 学生基础信息

统计各年级学生人数分布、男女比例等数据，以柱状图与环形图组合呈现学生结构。

### 5.1.5 活跃度时段统计

以面积图呈现一天内各时段的校园人流变化规律，反映校园活跃度的时间分布特征。

## 5.2 教学研究

围绕学校教学资源积累与教研成果进行呈现，涵盖教学资源库、教师课题、名师工作室等内容。中央区域为三维智慧教室场景。

### 5.2.1 教学资源

统计各类教学资源的分类情况，包括试卷、学案、教学设计、微课、课件、素材等资源数量，以矩阵树图呈现资源分类占比。

### 5.2.2 资源统计

汇总教学资源核心指标，包括资源总量、云试题数、云资源数、近期更新数等，以数字翻牌器呈现。

### 5.2.3 资源更新动态

以滚动列表形式呈现最新上传的教学资源，包括资源名称、上传时间、上传教师等信息。

### 5.2.4 教师课题

统计教师科研成果，包括课例数、公开成果数、在研课题数等指标，以数字卡片呈现。

### 5.2.5 课题项目

以卡片轮播形式呈现在研课题项目，包括课题名称、负责人、课题状态等信息。

### 5.2.6 名师工作室

以卡片网格形式呈现学校名师工作室资源，包括工作室名称、主持人、成员数、成果数等。

## 5.3 行政办公

整合通知公告、值班安排、校历日程、考勤统计、会议管理等行政办公数据，反映学校行政管理运转状态。中央区域为三维行政办公区域场景。

### 5.3.1 办公概况

汇总行政管理基础数据，包括部门数量、教职工总数、今日出勤率、功能室数量等，以数字翻牌器呈现。

### 5.3.2 通知公告

以滚动列表形式呈现学校重要通知和公告，包括通知标题、发布部门、发布时间、通知类型等。

### 5.3.3 值班安排

以信息卡片形式呈现当日值班人员安排，涵盖行政值班、教师值班、安保值班等岗位人员信息及联系方式。

### 5.3.4 校历日程

以热力图与时间轴结合的形式呈现学校重要活动和日程安排，包括本周重要活动、近期考试安排、校园活动预告等。

### 5.3.5 教职工考勤

统计教职工考勤情况，包括今日出勤、请假、缺勤人数。以柱状图呈现各部门出勤率对比，折线图呈现月度出勤趋势。

### 5.3.6 会议管理

以表格列表形式呈现学校会议安排和会议室使用情况，包括今日会议数、本周会议数、会议室使用状态等。

## 5.4 智慧图书

整合馆藏资源、借阅统计、热门图书、班级借阅排行、阅读活动等数据，反映图书馆资源与阅读服务情况。中央区域为三维图书馆场景。

### 5.4.1 馆藏概况

汇总图书馆馆藏资源数据，包括纸质图书总量、电子图书数量、期刊种类、报纸种类等，以数字翻牌器呈现。

### 5.4.2 借阅统计

统计图书借阅服务数据，包括今日借阅量、今日归还量、在借图书数、逾期图书数等，以数字卡片呈现实时数据，折线图呈现近期借阅趋势。

### 5.4.3 热门图书

分析图书借阅偏好，以横向柱状图呈现借阅量排行榜，饼图呈现各类别图书借阅占比。

### 5.4.4 班级借阅排行

以横向柱状图呈现各班级借阅量排名及年级借阅对比。

### 5.4.5 阅读活动

以卡片轮播形式呈现图书馆阅读推广活动，包括读书节活动、阅读分享会、好书推荐等。

### 5.4.6 入馆统计

统计图书馆使用情况，包括今日入馆人数、实时在馆人数等，以柱状图呈现入馆时段分布。

## 5.5 智慧教学

整合课程安排、教室使用、学生出勤、考试管理、班级管理、教学设备等教学运行数据，反映教学过程实时状态。中央区域为三维教学楼场景，以热力图形式呈现教室使用情况。

### 5.5.1 教学概况

汇总教学运行核心数据，包括今日课程数、正在上课数、教室总数、教室使用率等，以数字翻牌器呈现。

### 5.5.2 课程安排

统计课程分布情况，包括各年级今日课程数、各学科课程分布、课程时段分布，以柱状图、饼图、热力图组合呈现。

### 5.5.3 教室使用

反映教室空间资源使用情况，包括使用中教室数、空闲教室数、各楼栋教室使用率、教室类型分布等。三维场景中以颜色标注教室使用状态，形成空间热力效果。以进度环与3D热力图组合呈现。

### 5.5.4 学生出勤

统计学生出勤情况，包括今日出勤率、各年级出勤情况、各班级出勤排名、出勤趋势等。以仪表盘呈现整体出勤率，柱状图呈现班级排名，折线图呈现出勤趋势。

### 5.5.5 考试管理

呈现考试安排和成绩概况，包括近期考试安排、本学期考试次数、各年级平均分、成绩分布等。以表格列表与柱状图组合呈现。

### 5.5.6 班级管理

以表格列表形式呈现班级基本信息，包括班级总数、各年级班级数、班主任信息、班级人数等。

### 5.5.7 教学设备

反映教学设备运行状态，包括设备总数、在线设备数、离线设备数、故障设备数、设备类型分布等。三维场景中标注设备位置与运行状态。以数字卡片、环形图与3D场景标注组合呈现。

## 5.6 智慧安防

整合视频监控、门禁管理、学生请假、访客管理、告警事件、食堂安全等安防数据，反映校园安全态势。中央区域为三维校园场景，以点位标注形式呈现监控摄像头、门禁设备分布。

### 5.6.1 安防概况

### 5.6.3 门禁管理

### 5.6.4 学生请假管理

### 5.6.5 访客管理

### 5.6.6 告警事件

### 5.6.7 食堂安全

# 第7章 测试与验收

## 7.1 测试环境与内容

可视化效果测试在真实大屏硬件环境中进行，测试环境由校方提供大屏硬件设备，我方完成系统部署与调试。测试内容包括：

- **画面效果测试**：验证大屏画面与视觉设计稿的一致性，包括布局、配色、字体、图表样式等。
- **三维场景测试**：验证三维场景在大屏上的渲染效果，包括建筑模型、贴图质量、灯光效果、镜头动画等。
- **数据展示测试**：验证各数据面板的指标数据能否正常显示和更新，图表交互是否正常。
- **交互功能测试**：验证场景点击、数据钻取、专题切换、弹窗展示等交互功能的正常运行。
- **整体运行测试**：验证系统在大屏环境下的整体运行稳定性，包括长时间运行、页面切换等场景。

## 7.2 验收标准

- 大屏画面效果与视觉设计稿保持一致。
- 三维场景建模内容与本规格书约定的建模清单一致，建模精度满足约定要求（外观轮廓偏差不超过1米）。
- 各数据面板指标能正常显示，数据可正常刷新。
- 各项交互功能可正常使用：360度展示、点选缩放拖拽、楼栋房间聚焦跳转、设备分层、课室聚焦、UI弹窗信息展示等。
- 系统在大屏环境下可稳定运行。
