# Playwright E2E Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Playwright E2E tests covering core demo flows: page load, 6 topic navigation, 3D building click, UI theme switch, alert popup, responsive layout.

**Architecture:** Standalone `e2e/` directory with Playwright config. Chromium only (WebKit/FF optional). Uses Vite dev server via `webServer` config. MSW mock data active by default in dev mode.

**Tech Stack:** Playwright, Chromium, Vite


- [ ] **Step 2: Create Playwright config**

Write `e2e/playwright.config.ts`:

```ts
import { defineConfig } from '@playwright/test'

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: 'list',
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
  },
  webServer: {
    command: 'pnpm dev',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 30000,
  },
})
```

- [ ] **Step 3: Add e2e npm script**

Read `package.json`, add to "scripts":
```json
"test:e2e": "playwright test --config=e2e/playwright.config.ts"
```

Expected: Version string

```ts
import { test, expect } from '@playwright/test'

test.describe('UI Theme Switch', () => {
  test('should toggle between dark and light mode', async ({ page }) => {
    await page.goto('/')
    // Default is dark mode
    await expect(page.locator('html')).toHaveAttribute('data-ui-theme', 'dark')

    // Click the UI theme toggle button
    await page.getByText('☀️ 亮色').click()
    await expect(page.locator('html')).toHaveAttribute('data-ui-theme', 'light')

    // Toggle back
    await page.getByText('🌙 暗色').click()
    await expect(page.locator('html')).toHaveAttribute('data-ui-theme', 'dark')
  })
})
```

### Task 3: Write topic navigation + building interaction tests

**Files:**
- Create: `e2e/tests/topic-navigation.spec.ts`
- Create: `e2e/tests/building-interaction.spec.ts`

- [ ] **Step 1: Write topic navigation test**

Write `e2e/tests/topic-navigation.spec.ts`:

```ts
import { test, expect } from '@playwright/test'

test.describe('Topic Navigation', () => {
  test('should switch between all 6 topics', async ({ page }) => {
    await page.goto('/')

    const topics = ['教学研究', '行政办公', '智慧图书', '智慧教学', '智慧安防']
    for (const topic of topics) {
      await page.getByText(topic, { exact: true }).click()
      await page.waitForTimeout(500)
      // Verify topic-specific panel appears
      await expect(page.locator('text=总体概况').first().or(page.locator('text=办公概况').first().or(page.locator('text=馆藏概况').first().or(page.locator('text=教学概况').first().or(page.locator('text=安防概况').first())))).toBeVisible({ timeout: 5000 })
    }
  })

  test('should keep scene canvas mounted after topic switch', async ({ page }) => {
    await page.goto('/')
    const canvas = page.locator('#root canvas').first()

    await page.getByText('行政办公').click()
    await page.waitForTimeout(500)
    await expect(canvas).toBeAttached()

    await page.getByText('智慧安防').click()
    await page.waitForTimeout(500)
    await expect(canvas).toBeAttached()
  })
})
```
