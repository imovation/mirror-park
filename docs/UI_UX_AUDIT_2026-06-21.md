## 二、通用问题 (跨 7 个专题)

### G1. TopMetrics 横条遮挡 3D 场景顶部 ⚠️ P0

**现象**: 7 个专题的 TopMetrics 都漂浮在 3D 场景顶部 (z-index 在场景之上),导致:
- 综合态势 6 个 metric 挡到崇文楼/崇德楼
- 教学研究 5 个 metric 挡到崇德楼
- 智慧教学 5 个 metric 挡到崇文楼/崇智楼
- 智慧后勤 5 个 metric 挡到崇德楼/崇智楼

**根因**: TopMetrics 容器 `position: absolute; top: 0` 放在 `<Canvas>` 外的 UI 层 (z-index:1),与 3D 场景叠加

**优化方案**:
- 方案 A (推荐): TopMetrics 内嵌到 ScreenLayout 的顶部条,作为 Header 的一部分 (高度 48-56px)
- 方案 B: 缩短 TopMetrics (50px → 36px) 并加半透明背景,在保留悬浮感的同时弱化遮挡
- 方案 C: 切换主题时短暂 fade out,完成后 fade in

### G2. 小型 BarChart 高度不足 ⚠️ P1 (影响 4+ 专题)

**现象** (出现在多个 panel):
- 教学研究 - 资源类型分布 (3 条 bar,高度 ~50px)
- 教学研究 - 资源统计 (3 条 bar,高度 ~50px)
- 智慧图书 - 图书借阅排行 TOP10 (10 条 bar,高度 ~50px)
- 智慧图书 - 班级借阅量排行 (8 条 bar,高度 ~50px)
- 智慧教学 - 年级排课分布 / 各楼宇使用率 (3 条 bar)
- 智慧教学 - 各年级出勤率 (3 行,2 列)
- 智慧教学 - 年级均分 / 成绩分布 (3-4 条)
- 智慧后勤 - 各餐次就餐人数 (3 条)

**根因**: Panel 高度被 flex-1 (1份) 分配,但内部又塞了 2-3 个组件,BarChart 只能分到 50-80px

**优化方案**:
- BarChart 内部: `min-height: 120px` 强制最小高度,或
- Panel 内部布局: 子组件用 `flex` 比例分配,BarChart 至少 `flex: 1.5`
- 短列表 (≤4 条): 横排 BarChart (2-3 列 grid) 替代单列竖排

### G3. 文字溢出/截断 ⚠️ P1 (影响 3 个专题)

**现象**:
- 教学研究 - 名师工作室: "数学创新教学工..." 截断
- 教学研究 - 课题项目: "高中语文整..." 截断
- 智慧教学 - 出勤排名与空间使用: 班级名"初一(8)班"等挤在左侧

**根因**: Grid 卡片 `text-overflow: ellipsis` 触发,源文本超过容器宽度

**优化方案**:
- 2-3 行布局: `display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical`
- 文字过长的 panel: 缩字号 (`font-size: 12px → 11px`) 或缩短文本 (教研简报 → 教研)

### G4. 7 专题配色一致但区域色过度 ⚠️ P2

**现状**: 每个专题定义主色 + 8 个语义色 (CHART_PALETTE),12 色调色板已统一
**问题**:
- TopMetrics 6 个 metric 用前 6 种 chart palette 色,某些组合对比度过高 (橙+红)
- 部分 panel 内多图表用同一组颜色,辨识度下降

**优化方案**:
- TopMetrics 用主题色 (1 色) + 同色系 2-3 阶 (明/暗/淡)
- 同 panel 内多图表用色相轮转 (青/紫/橙/绿) 增强对比

## 五、风格/色彩系统优化

### 5.1 TopMetrics 配色统一

**现状**: 6 metric 用 chart palette 前 6 色 (青/紫/绿/橙/粉/红),某些组合过艳
**建议**: 1 主色 + 同色系 3 阶
```typescript
// 示例: 教学研究 (主色紫)
const TOP_METRICS_COLORS = [
  'var(--theme-primary)',        // 紫 (主)
  '#c084fc',                      // 紫 (浅)
  '#7c3aed',                      // 紫 (深)
  '#a78bfa',                      // 紫 (淡)
  '#ddd6fe',                      // 紫 (极淡)
  '#ede9fe',                      // 紫 (背景)
]
```

### 5.2 Panel 内多图表配色

**现状**: 同 panel 内的多图表使用同一组 chart palette,易混淆
**建议**: 相邻图表用色相轮转
- 图表 1: 主题主色
- 图表 2: 主题辅色 1 (主色 30° 色相)
- 图表 3: 主题辅色 2 (主色 -30° 色相)
- 图表 4: 强调色 (语义色: 成功/警告/危险)

### 5.3 字号阶梯

**现状**: 各 panel 字号不统一
**建议**: 定义 6 级字号阶梯
```css
--font-size-3xs: 10px;  /* 辅助标签 */
--font-size-2xs: 11px;  /* 图表副标题 */
--font-size-xs: 12px;   /* 卡片小字 */
--font-size-sm: 13px;   /* 列表 */
--font-size-md: 14px;   /* 正文 */
--font-size-lg: 16px;   /* 小标题 */
--font-size-xl: 20px;   /* 大数字 */
--font-size-2xl: 28px;  /* 标题数字 */
--font-size-3xl: 36px;  /* 巨型数字 */
```

## 六、布局密度建议

| Panel 类型 | 子组件数 | 推荐 panel 高度 |
|-----------|---------|---------------|
| 单图表 | 1 | flex-1 (1份) |
| 数字 + 图表 | 2 | flex-2 (2份) |
| 数字 + 2 图表 | 3 | flex-3 (3份) |
| 数字 + 2 图表 + 列表 | 4 | flex-3+ (3份+滚动) |

**密度规则**:
- 子组件 ≤2: flex-1 / flex-2 即可
- 子组件 ≥3: 必须 flex-2 以上,且启用 panel-scroll
- 长列表 (≥5 条): 独立 panel 占满一栏,不要与图表混排
